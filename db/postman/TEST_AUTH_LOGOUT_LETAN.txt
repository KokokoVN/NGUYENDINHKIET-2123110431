====================================================================
TEST AUTH + LOGOUT (letan) - Postman
====================================================================

Base URL:
  http://localhost:5066

--------------------------------------
1) LOGIN LETAN (lấy token)
--------------------------------------
Method:
  POST
URL:
  http://localhost:5066/api/auth/login
Header:
  Content-Type: application/json
Body (raw JSON):
{
  "username": "letan",
  "password": "123456"
}

Kỳ vọng:
  - Status 200 nếu đăng nhập đúng
  - Response có accessToken (dùng cho bước logout)

--------------------------------------
2) LOGOUT LETAN (ghi AuditLog)
--------------------------------------
Method:
  POST
URL:
  http://localhost:5066/api/auth/logout
Header:
  Authorization: Bearer <accessToken>

Body:
  (không cần body)

Kỳ vọng:
  - Status 200: {"message":"Đăng xuất thành công."}

--------------------------------------
3) (Tuỳ chọn) XEM LOG TRONG SQL
--------------------------------------
Chạy query:
SELECT TOP 20
  AuditId, CreatedAt, Action, EntityName, EntityId, ActorUserId, Reason
FROM dbo.AuditLog
WHERE Action IN ('AUTH_LOGIN_SUCCESS','AUTH_LOGIN_FAILED','AUTH_LOGOUT_SUCCESS')
ORDER BY AuditId DESC;

====================================================================
GHI CHÚ
====================================================================
- Nếu LOGIN bị 401 (Unauthorized) thì sẽ không có token để gọi logout,
  vì vậy sẽ chỉ có log AUTH_LOGIN_FAILED (không có AUTH_LOGOUT_SUCCESS).
- API logout là endpoint không có "thu hồi token" vì JWT không stateful;
  hệ thống chỉ ghi log và bạn xoá token ở client.
